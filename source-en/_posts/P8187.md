# 1. Problem Statement
Given $n$ two-dimensional vectors, for every $k$ ($1\le k\le n$), count the ways to choose $k$ vectors whose sum is $(x_g,y_g)$.

# 2. Analysis
## 2.1 Brute Force
Enumerating all subsets costs $2^n$ and is impossible for $n=40$.

## 2.2 Meet-in-the-Middle with a Map
Split the vectors into two halves. Enumerate every subset of each half and store `(sum_x, sum_y, k)` and its frequency. For every first-half state, look up the complementary second-half state `(x_g-sum_x, y_g-sum_y, k_total-k)`. `unordered_map` is preferable to `map` because the expected lookup is $O(1)$ rather than $O(\log n)$.

## 2.3 Meet-in-the-Middle with Two Pointers
Store the states in two vectors and sort by `x`, then `y`, then `k`. Start with the smallest first-half state and largest second-half state. If the sum is too small, advance the first pointer; if too large, decrease the second pointer. When equal, count the complete equal runs and combine their frequencies by `k`.

An alternative groups states by `k` first (`vector<Instruct> fir[20], sec[20]`), then runs the two-pointer scan for every pair of counts. This uses less memory because `k` is represented by the array index instead of being stored in every state, though it is slightly slower.

## 2.4 An Unusual State Enumeration
Each vector is selected or not selected, so a subset can be represented by a binary number. Enumerate integers from 0 to $2^{n/2}$ and inspect their bits instead of using DFS. This is mainly a curiosity; DFS is simpler in a contest.

# 3. Complete Code

The essential implementation is:
```cpp
#include <bits/stdc++.h>
using namespace std;
using ll = long long;
struct Instruct {
  ll x, y; int k;
  bool operator<(const Instruct& b) const {
    if (x != b.x) return x < b.x;
    if (y != b.y) return y < b.y;
    return k < b.k;
  }
  bool operator==(const Instruct& b) const { return x == b.x && y == b.y && k == b.k; }
};
int main(){
  int n, target_x, target_y; scanf("%d%d%d", &n, &target_x, &target_y);
  vector<pair<ll,ll>> a(n);
  for(auto& [x,y]:a) scanf("%lld%lld", &x, &y);
  int half = n/2;
  vector<Instruct> left, right;
  for(int mask=0; mask<(1<<half); ++mask){
    ll x=0,y=0; int k=0;
    for(int i=0;i<half;i++) if(mask&(1<<i)){x+=a[i].first;y+=a[i].second;k++;}
    left.push_back({x,y,k});
  }
  for(int mask=0; mask<(1<<(n-half)); ++mask){
    ll x=0,y=0; int k=0;
    for(int i=0;i<n-half;i++) if(mask&(1<<i)){x+=a[half+i].first;y+=a[half+i].second;k++;}
    right.push_back({x,y,k});
  }
  sort(left.begin(),left.end()); sort(right.begin(),right.end());
  vector<ll> ans(n+1);
  for(size_t i=0,j=right.size(); i<left.size() && j; ++i){
    while(j && (left[i].x+right[j-1].x>target_x ||
      (left[i].x+right[j-1].x==target_x && left[i].y+right[j-1].y>target_y))) --j;
    for(size_t t=j;t<right.size() && left[i].x+right[t].x==target_x && left[i].y+right[t].y==target_y;t++)
      ans[left[i].k+right[t].k]++;
  }
  for(int k=1;k<=n;k++) printf("%lld\n",ans[k]);
}
```

The two-pointer variants have the same theoretical complexity because every state is traversed at most once. The hash-table implementation can be found in the linked submission, but its hash function in the original experiment was not fast enough.
